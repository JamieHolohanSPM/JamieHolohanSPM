Process for handling Level (L3) CRM Support

The CRM support system is tracks customer issues internally within the company. The customer uses a separate system to raise issues known as PMRs in the Retain system. Level 2 (L2) Support create CRM to track the issue internally. So each CRM is associated with a PMR. If a fix is going to be delivered to a customer by L3, then an APAR will be created against the CRM and published to the associated CRM so that the customer can track the fix using that APAR number. This process documented below describes how the CDEJ team handle CRMS that are assigned to them. 

 

Response times for CRM assignees:

CRM are expected to update an assigned CRM with an update on progress within a specified period of time depending on severity. See below for response times(after the CRM is assigned to assigee) that are mapped to severity.

  • Critsit: This means any CRM raised by a certain customer is production critical so the response time is Close of Business
  • Sev 1: Close of Business 
  • Sev 2: 2 days 
  • Sev 3: 1 week
  • Sev 4: 2 weeks

 

CRM Lifecycle:

  • L3 - Investigating
  • Waiting for L2 Action
  • Waiting for Customer Action
  • Pending Closure

Gatekeeper Role for CDEJ Process:

Each week there is an assigned gatekeeper (link above) whose role is to triage unassigned team CRMs and assign them to individual team members so that they can be investigated, progressed and ultimately closed.

CRMS with highest severity should take precedence over others when considering how to handle those in the backlog (current queue). 

 

New Escalated CRMs

Before assigning a CRM, the gatekeeper should consider the following:

  1. Has the CRM the requested information on it: see here for a checklist of information that should be present. If this information is not present then it should be assigned back to L2 support to provide these details.
  2. Which category does the CRM belong to; defect, query, enhancement? If it is the latter two then there are some standard responses that can be applied to the CRM here (https://w3-connections.ibm.com/wikis/home?lang=en-us#!/wiki/W95a3462db6e6_4c78_9651_aeb7ceafd9de/page/Standard%20Responses) and the CRM can be set to 'Pending Closure'.
  3. If it is a defect and the required information has been provided you should check the issue against a checklist of known issues where we can provide advise and guidance and advice that will help the customer to resolve them. This checklist can be found here:
  • https://w3-connections.ibm.com/wikis/home?lang=en-gb#!/wiki/W0c6659bf1ea2_4ec4_a931_2222e7accb81/page/FAQ%20Notes/ - Client FAQ and Common Support Cases.doc
  • https://w3-connections.ibm.com/wikis/home?lang=en-gb#!/wiki/W95a3462db6e6_4c78_9651_aeb7ceafd9de/page/FAQ%20Collateral - FAQ Collatoral

         4. If the new CRM is ; a defect, has all the required information required on it and is not a known issue, then it should be assigned out to a team member and Curam-TI CDEJ-WW should be added to the CC list.

         5. Add the following token to the CRM: <CDEJ_ACCEPT_ASSIGNEE_FNAME>, where 'ASSIGNEE_FNAME' is the first name of the assignee. So for example if the assignee will be Sharunas then add the following token: <CDEJ_ACCEPT_SHARUNAS>. It would also be a good idea to let the assignee know (via Sametime) that you have assigned or are about to assign a CRM to them .

Existing CRMs

The gatekeeper should review existing CRMs with assignees and try and progress them along so that they can be closed.

 CRM Summary

Gatekeeper should sent handover email at the end of the week with summary stating the following:

  • Num of S1 CRMs with CDEJ
  • Num of S2 CRMs with CDEJ
  • Num of CRMs currently being investigated.

CRM assignee for CDEJ Process:

When a CRM is assigned to a team member (assignee) for investigation the status of the CRM should be set to 'L3 - Investigating'. The objective of the assignee is to find the source of the issue so that the CRM can be be set to 'Pending Closure' and both the CRM and associated PMR can be closed. 

 

There is alot of activity on CDEJ CRMS. If the assignee wants to be notified of the CRMS assigned to them separately from the other nes that they are CC'd on. They can set up a filter in Lotus Notes as follow:

(1) From 'Actions' Menu in Notes , select 'New Rule'

(2) Create the New Rile as follows:

 (i) Specify the Condition as: When Sender contains CDEJ_ACCEPT_NAME, where 'NAME' is your first name.

 (ii) Specify the Action to take by specifying the folder these CRMS should be moved to.
