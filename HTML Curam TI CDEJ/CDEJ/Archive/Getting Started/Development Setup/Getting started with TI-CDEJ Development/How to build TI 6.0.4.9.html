How to build TI 6.0.4.9

Overview

  • These guidelines allow to build the TI 6.0.4.9 and set up WebSphere;

  • Note: Windows is required. Follow Getting a Virtual Machine to get the Windows enviroment

 

Set up the development enviroment and the application

Follow these instructions to cretate a new development environment for your snapshot view:

  •   Create a file named setenv.bat with the JAVA_HOME variable that contains the location of your JDK folder (such as set JAVA_HOME=C:\Program Files\Java\jdk1.8.0_201), and save it  at C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI
  • Run setenv.bat
  • From C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI\devenv\setup, run: setup.bat
  • Update setenv.bat:
    1. Set the JAVA_HOME, this variable must point to the java folder inside the devenv folder (such as C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI\devenv\tools\java)
    2. Set the ANT_HOME variable according to the ant folder inside the devenv (e.g. C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI\devenv\tools\ant
    3. Include the ANT_HOME in the path variable, use the following code: set path=%ANT_HOME%\bin;%PATH%
  • From C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI, run setenv.bat
  • From C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI, run install_appserver.bat was
  • Update setenv.bat as shown below:
    1. Update the JAVA_HOME, this variable must point to the java folder inside websphere (such as C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI\devenv\tools\websphere\java)
    2. Define WAS_HOME, this variable points to the websphere folder (e.g. set WAS_HOME=C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI\devenv\tools\websphere)
    3. Define J2EE_JAR, this variable stores the location of the jee.jar, which is inside the lib folder in websphere (e.g. set J2EE_JAR=C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI\devenv\tools\websphere\lib\j2ee.jar
    4. Set up the coreinf\properties\Bootstrap.properties and coreinf\properties\AppServer.properties files. Create these files by copying the sample files located in the same directory and editing them as appropriate. Set up Websphere

Prerequisites: all the steps above are required with the exception of the ones about the eclipse configuration.

Note: This steps show how to set up Websphere and deploy ClientMockApp on it.

  1. From C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI, run setenv.bat
  2. Run build.bat configure *
  3. Run build.bat clientmockapp-build-deploy-websphere, this command allows to build and deploy the ClientMockApp on Websphere. We can deploy on Websphere other project using the following commands:
    1. build.bat wasclientdeploy : Deploys the test client application on Websphere
    2. build.bat wasdeploy : Deploys the in-container test application on Websphere
    3. build.bat wasextclientdeploy : Deploys the test client external application on Websphere
    4. build.bat wastabtestruntests : Deploys and runs tests on the Tab Test Application for WebSphere; Undeploying when finished
    5. build.bat wastabtestrunwebspectests : Deploys and runs tests on the Tab Test Application for WebSphere. Undeploying when finished

  4. Access the application at http://localhost:9080/ClientMockApp/

* If you have some errors while running this command you may already have a Websphere profile or an instance of Websphere is running:

  1. From C:\Users\Administrator\workspace\TI-Integration-6.0.4.9_iFix5\TI\deveng\tools\websphere\bin, run stopServer.bat CuramServer
  2. Remove the Websphere profile, run manageprofiles.bat –delete –profileName AppSrv01
  3. Try again to run build.bat configure inside the TI folder.
