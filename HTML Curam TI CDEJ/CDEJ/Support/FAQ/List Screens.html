Common Resolutions

In response to long lists performance complaints, especially when compared to 5.2:

Long list pages in Curam, especially thousands of items, will always have performance problems. This was the case in Curam 5.2 and such pages are not recommended. In Curam 6.0 signification changes have been made to the UI to add new dynamic features, e.g. expandable rows and list row items. These features have added to the loading time for list pages, however in addition pagination support has been provided. Pagination support improves the loading time of list pages by ensuring only the first set of items to be displayed are rendered in the HTML.

However, long lists are not recommended and will encounter performance issues, especially in older versions of Internet Explorer, such as IE7 and IE8.

The recommendations are to redesign such pages to include a search functionality and reduce the number of items returned where they exceed an amount. For example, the readmultimax feature can be used for this.

<INTERNAL> Where the page being complained about is OOTB, the case should be directed at the team owning the page to redesign and handle issue for customer.
