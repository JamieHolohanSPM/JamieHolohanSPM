# Issue from user perspective Details Affects Known Since Fixed On JIRA CRM APAR
1 IE specific: document doesn't open, main app is greyed-out and the little popup stays on screen

Caused by sporadic Jacob Exception (essentially a NPE) when opening a document, happens when some IE windows get closed or otherwise invalidated while our code is looping over them to find the Cúram window.


Fixed by defensive checking for nulls in the loop.

 6.0 SP2 + 29/10/2014   TEC-17949 44571,021,724 PO04285
2 Changes to Word document intermittently not saved and "page expired" errors

Timing issue, trying to save while previous save is still in progress. For example if user hits Save twice in a row.


Fixed by checking for a save in progress when a save or exit is requested.

 6.0 SP2 + 11/11/2014           TEC-18006  84761,999,649 PO04338
3 Browser freezes when a Word document is opened or closed

Caused by different editing sesions sharing a JVM. There will be a Jacob error in the logs: "DLL already loaded in another classloader"

 

Fixed by enforcing a new JVM for every session.

 6.0 SP2 + 28/11/2013   TEC-15705 74304,999,649 78291,999,649 PO02392
