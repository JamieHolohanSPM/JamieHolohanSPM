Online Help and Client Infrastructure

Question:

What Client Infrastructure is present to generate the online help for the SPM application?

Answer:

The current status of this infrastructure is as follows:

1) The online help is always generated along with the application, unless switched off by the console switch (called DISABLE_HELP).

 

2) The help generation is integrated with the application (client) build process, and there is no separate target to build just the online help files any longer.
    The help metadata is processed along with the page properties when generating the pages for the application, and the help property files are saved by the client-persistence project classes, and then jarred along with the page properties files into the pageprops_<localecode>.jar libraries found under the WebContent/WEB-INF/lib.

 

3) The help properties for the fields on the page are ending up with .help, these are optionally included into any page associated .properties file and end up in the mentioned separate properties files used by the online help system.

 

4) The online help page for a page is displayed by a single handcrafted JSP page called help.jsp which is passed the page id. This retrieves some properties from the associated CDEJResources.properties file.
   The file uses curam.util.client.render.component.OnlineHelpRenderer to display the actual content for a help page, to wrap the properties and their texts into the tabular format.

 

 

Previously there was either help applet or help servlet used to display the help pages and a separate target to generate help pages, the applet solution stopped to be used earlier, the servlet does not seem to be present in web.xml either.


Also, it might be that there is couple of left-over artifacts from the older online help system to remove when tidying up.
