Overview

This page and the containing pages contain information about the current ongoing and past completed designs within the CDEJ component. They are organised by release so that it can be easily seen at a glance which ones have been completed and released in the product.

IBM Design Thinking

Any big features in a normal release (fix pack) need to go through the design thinking process. In a major release like V7 this will apply for whole application.

See more information here: https://design.ibm.com/thinking.html

Think 40 Training

Keep up to date with IBM training and new emerging technology : https://think.atlanta.ibm.com/

Design Documentation

All new features and significant changes to Curam require a design page and review. Depending on the scale of the changes, different review groups should be engaged.

Large Scale Projects

For large scale features and new functionality the Architecture Review Process should be followed, including producing an Architecture Description Plan and Architecture Description document. Details of the full Architectural Review Process can be found here.

For example: The Mobile SDK Architecture Review

The full architecture documentation should be detailed on the main TI Architecture site here and should following the TI design template.

Small to Medium Infrastructure Features

Small to medium infrastructure features or significant reworking to a particular area also require design documentation.

The design tempate for these can be found here.

A design page, following this template,  should be added as a child of this page or it can be added to the design pages listed for the TI Architecture board. This is up to the developer, but the rule of thumb is that smaller changes can be added here, where they are being peer reviewed within the team. Larger changes that need a review across the infrastructure teams should be added to the TI Arch Board site.
