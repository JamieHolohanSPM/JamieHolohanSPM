1. Introduction

The Cúram IBM (Cúram) FILE_EDIT widget allows a user to launch Microsoft Word, edit the document and save the document directly to the Cúramdatabase. This functionality relies on instantiating a Java applet [1] to communicate with the launched Microsoft Word document. This use of a Java applet relies on the Java plugin being installed in the end user's browser. The Java plugin relies on the Netscape Plug-in (NP-API) [2] to function.

 

Google Chrome (Chrome) have made the decision to end support for NP-API in 2015 [3].  This effectively means that the Java plugin will stop working with Chrome 45, in September 2015 [4]. As a consequence the Cúram FILE_EDIT widget will cease to work when an end user upgrades to this version of Chrome.

 

Many Cúram customers have come to depend on this FILE_EDIT widget feature as part of their daily business process. With this in mind an alternative, applet-less, solution has been developed for Google Chrome. This alternative solution makes use of Google's recommend best practices [5].

2. Cúram applet-less solution for Google Chrome

Cúram have developed a solution based on a Chrome Extension & Native Messaging bridge to allow the Cúram application to interact with Microsoft Word.

 

Customers wishing to enable this feature for their end users must install two separate software components onto the computers of their end users:

1. Chrome Extension

2. Chrome Native Messaging Bridge

 

Please note that the feature cannot be used until both components are installed.

3. Installing required software

3.1 Installing the Chrome Extension

Cúram provide the Chome Extension as a .crx file. Since Chrome 33,  2013, Google have placed a restriction on the location that users can install extensions from [6]. Since this release, only extensions published to the Chrome Webstore [7] can be installed by users. Licensing restrictions prevent Cúram from publishing software in this way so an alternative installation method must be used. The recommended approach is to install the provided Chrome Extension is via the Chrome supported enterprise policy [8].
