Introduction

The FILE_EDIT widget is a UIM component that allows interactive editing of Microsoft Word documents directly from the Cúram application, without the need to save document files locally and re-upload after making changes.

 

While the feature is highly appreciated by customers, it is technically challenging to implement and as a result the current solution is not as reliable and secure as it should be. Improving reliability and maintaining/supporting the solution in the long term is very costly and given the circumstances of the use case (Cúram code interacting with components on the actual individual user machines, outside of our control) it is not even possible to achieve fully satisfactory implementation with a justifiable investment. Moreover, browser vendors are now moving away from NPAPI, which is required for the existing solution to work.

Given the above we have decided to provide a replacement, which would have the necessary functionality while avoiding all the issues.

Requirements

  • create professionally looking documents
  • intuitive creation of templates, e.g. WYSIWYG, directly in Word
  • automatically fill in data from DB to predefined places in template
  • output to common formats, PDF, Word, OpenOffice
  • templates can vary based on conditions

Use Cases

Can we come up with a use case where lack of interactivity would be a blocker or would at least seriously limit applicability?

Case Worker

  • send a predefined letter to a specific client, filling in some client data
  • send an ad-hoc/free-form letter to a specific client, filling in filling in some client data
  • "clause search" scenario in Appeals, where user makes search in Cúram and then manually picks a result to be added to specific place in the document
  • send a dynamically generated letter, e.g. where the content is largely determined by variables, as opposed to mostly predefined content

Admin

  • add a new document template
  • edit a document template
  • remove document template

Competitor Offerings

Here we examine what are the competitor capabilities in the area.

  • Windward DocGen: http://go.windward.net/docgen8.html
    • templates defined in Word or Excel by non-developers
    • custom data sources
    • output to lots of formats
    • conditional logic to dynamically show/hide parts of documents, generate lists, etc.
  • IC Template-Based Document Creation: http://ibmurl.hursley.ibm.com/MZZH
    • templates defined in Microsoft Word/Excel, Adobe Acrobat
    • data from enterprise DB
    • outputs to PDF, DOC, PPT, XLS
    • defines document management process (what if we need to re-generate a doc, change the template, etc.)

Existing Solutions within Cúram/IBM

  • IBM Rational Publishing Engine: http://ibmurl.hursley.ibm.com/MZZI
    • supports data sources
    • provides a graphical template editing environment for custom report design
    • supports multiple output formats and concurrent document generation to multiple target formats from a single template
  • Cúram XML server: http://ibmurl.hursley.ibm.com/MZZE
    • data in XML format provided by Cúram server
    • document template defined as XSL-FO (can be generated from a RTF document with third party software and tweaked)
    • output to PDF, RTF, HTML, or plain text
