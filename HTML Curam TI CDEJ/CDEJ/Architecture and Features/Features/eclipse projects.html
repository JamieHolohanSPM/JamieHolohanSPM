plug-in-framework (client/components/plug-in-framework) client Contains the client-specific domain plug-in framework implementation. 
There are several types of domain plug-in, each type performs a particular kind of processing.
The client infrastructure associates domain plug-ins with domain definitions (defined in the UML model) and then invokes these plug-ins at key points in the processing of the application data. These associations between domain plug-ins and domain definitions can be overridden as required and custom plug-ins can be written to effect the required custom processing.

duim-generator (commons/duim-generator) client

Contains the implementations for the duim project.

Dynamic UIM as its name implies, is UIM that is cached in the resource store - rather than static UIM (described in earlier sections) which resides on the file system - so that the server and client do not have to be rebuilt in order for a page to be displayed in an application. All string values in dynamic UIM documents must be externalized in properties files, which must also be cached in the resource store.

post-processing (reltools/glbntools/post-processing) both Takes input of the translated content and DNT zip processes these based on an input language and optional country.
This results in a Platform and CCS Language Pack zip ready for use to build the application with.
pre-processing (reltools/glbntools/pre-processing) both Takes an input CCS VOB and extracts the relevant content to form 3 deliverables:
- A TC.zip which contains the artifacts appropriate for translation
- A AC.zip which contains the artifacts appropriate for manual extraction
- A DNT.zip which contains all non-translated content in regards LOBs, DMX files, CTX and Messages files. This is used during post-processing to form the complete Language Pack in a Locale.
policing (reltools/policing) both  What is the Policing Tool?
 - A number of tools exist to check the quality of code in Cúram. One such tool is Policing. 

Policing tool consists of Policing delivered as standard by the JDE (e.g. checksql, validateallrulesets and validateallworkflows) and the Policing Tool which provides internal checking.

Other streams can write custom tools.
