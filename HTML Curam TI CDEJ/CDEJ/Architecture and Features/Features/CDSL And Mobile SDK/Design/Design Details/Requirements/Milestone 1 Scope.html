FEATURES:

  • Cúram Data Store Implementation (curam.util.dataservice.CRUDStore)

An implementation of the Dojo DataStore interface, with Cúram specific features to allow you to hook it up to a Cúram "facade" class.
NB: The first implementation will accept the facade name directly. This will be replaced by a logical name in future when we make the service configurable in a future drop (i.e. "ObjectivesService" -> curam.facade.impl.ObjectivesFacade"). This configuration will also serve as a white list of operations. I guess all we're saying is, there is still some details to be worked out in this area, you may have to change the parameter(s) that you pass to the "constructor" of the Cúram data store in future drops.
We will support the following Dojo Store interface methods:

  • get
  • query
  • put
  • add
  • remove
  • getIdentity

The methods of the store interface will be mapped by convention to methods of the Cúram facade e.g. Store.remove -> ObjectivesFacade.remove i.e. you will have to create your facades in a certain way. (Question: you are prototyping the user interface, has anyone begun building the server code\facades, if so we my have to add additional features here)

  • Cúram RPC API (curam.util.dataservice.RPC)

Invokes Cúram facades outside the context of a Data Store. (e.g. RPC.call(facade, inputJson).then())
We need to implement this to use internally in the implementation of "1" above. So, we could make it "public" in the first drop. If you think you'll be happy with just the store, then let us know, we'll leave it out. It'll save us some work as we could take shortcuts in the internal implementation of "1", save some documentation etc.

  • JSON Data Structure

A definition of the JSON to Cúram "struct" mapping
Support the following Cúram data types:
String
Integer types 8-bit to 64-bit
Note on 64-bit integer: Since it's not supported natively in JavaScript, this will be serialised as a string. No additional support\helper methods will be provided for 64 bit arithmetic.
Floating Point (float + double)
Money
Code table
Date + DateTime
A wrapper API (curam.util.dataservice.Struct) for accessing the JSON structure. Although we will document the JSON data format, you will access values through a wrapper API. (i.e. that may eventually apply "fixups" etc). We feel we should introduce this in the first cut, so that we avoid impacting your code as much as possible as additional drops are taken on.

  • Error Handling

Errors generated from the facade invocation will be handled through the standard callback methods. It will be up to you to what you do with the exception and it's associated message(s) (e.g. business validations, cross-field validations from the server).
Error handling for things like "timeouts" will just be "whatever Dojo does by default".

  • Documentation + Sample Widget

Goes without saying we'll be producing some material to show ye how to use the API, plus there will be a sample widget which uses the features (e.g. A grid (gridX) backed by the Cúram store).
