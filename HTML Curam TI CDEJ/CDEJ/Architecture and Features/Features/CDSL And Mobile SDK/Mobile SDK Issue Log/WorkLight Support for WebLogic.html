WorkLight Support for WebLogic

Description:

Aligning to mobile first strategy means relying on MCoC Core Framework and WorkLight as part of the mobile stack.

  • WorkLight is not supported running on WebLogic
  • WebLogic customers therefore will not get the same collateral.
    • Most notably the Sample Application, which provides significant starting point for mobile applications, in this use case is unusable by WebLogic Customers.
  • WebLogic customers potentially early adopters (e.g. MSD)


To support WebLogic customers we would have to create a second WorkLight independent sample application.

  • Adding significant development effort to the project
  • Diverging from the IBM mobile first strategy
  • Reducing the value/need to a WebSphere customer to pay 120k WorkLight licence per app deployment.
  • Need view on the WorkLight toolkit  key advantages, are they compelling enough for websphere customers to adopt?

Note: This issue has implications for our choice on integrating with MCoC Core Framework.

Status: Closed

Assigned To: Eamonn

Resolution: Feedback from playback 0 is to support bluestack only, i.e. no support for WebLogic.
