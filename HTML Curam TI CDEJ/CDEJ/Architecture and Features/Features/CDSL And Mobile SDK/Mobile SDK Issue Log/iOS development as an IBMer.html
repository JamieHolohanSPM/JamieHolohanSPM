iOS development as an IBMer

Description: What are the implications for an IBMer working in an iOS development environment?

MCoC Information around Licensing & Usage of mobile SDKs can be found here. This is specific to Software Group, but GBS are likely to have similar.

It is worth noting the following from this information:
Development of applications for IBM customers should be done under the customers' SDK licenses. The Apple iOS agreements in particular do not allow IBM to develop an application under its own license and then distribute that application directly to a customer.

Internally it is also worth noting that for Android and Apple we need to get a License via the SLM Portal.
We are following the necessary channels to add CÃºram to the projects listed as approved for Apple and Android and make it easier to arrange android/app licenses going forward.

For WorkLight apps, we can ship these as they are not built binaries, and customers can use them within WorkLight with the following caveats:

  • They cannot change the project structure.
  • They cannot process data by the mobile application that does not originate directly from the principal program (of IBM WorkLight Consumer Edition).
  • They cannot create new projects.
