Overview

  • provides an API that shows confirmation dialog if user is leaving the page
  • globally dynamically configurable with application config
  • developer can provide custom rules for when the dialog should not kick in
  • see release note on http://pdappsrv.mul.ie.ibm.com:8080/browse/TEC-13001

How it Works

  • it is a JS API
  • has an install() function, which enables the function for current page
  • by default it shows confirmation dialog when user navigates away from current page
    • handles mailto: links automatically
  • developer can install custom "condition handlers" to avoid the confirmation dialog in specific cases
    • AppExitConfirmation.registerExitConditionHandler()

JavaScript

  • curam.util.ui.AppExitConfirmation
  • curam.util.ui.AppExitConditionHandler

Application Config

  • curam.external.app.guard.against.leaving
  • note this is only provided OOTB in external app container - other app environments, including internal Tabbed UI - have to use that app property and do AppExitConfirmation.install() based on that
