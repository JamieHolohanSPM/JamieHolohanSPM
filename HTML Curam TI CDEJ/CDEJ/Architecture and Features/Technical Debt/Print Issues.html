Print Issues

 

 

A number of Customers have been opening support tickets about the print issues on the Caseworker application. Most of them have been complaining  that the print page does not print the same information that they can see on the screen. The print option has cut of information during the print page.

This kind of issues has been fixed on the caseworker application. However, two main problems have been highlighted that could not be fixed with the actual solution for print pages:

 

1) Content break on the margin:

  • Sometimes, we can see texts that have been breaking in the same point that the page have a break. In this scenario is it common  to see part of the content on the first page and the remaining text on the second page.
  • It is happening because of the paper margin.
  • Following the PDF  that demonstrates the scenario: pageBreaksEvidence

 

 

2) Extra blank space after the first expandable list

  • In some scenarios, the print preview shows a blank space after the first expandable list. This problem happens because the height defined for the iframe does not fit on the remaining page. So, the print browser break the page to fit the content on the next page.
  • The problem only happens when the user selects the layout "landscape" to print the page.
  • Following the PDF  that demonstrates the scenario: blankSpaceEvidence.pdf

 

In touch with your PO(Shane MCfadden), it was agreed that we accept these two problems.
