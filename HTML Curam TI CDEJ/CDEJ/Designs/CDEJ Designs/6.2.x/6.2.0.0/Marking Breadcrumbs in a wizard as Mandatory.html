Marking Breadcrumbs in a wizard as Mandatory

Background

This is part of the Guided Evidence Management project: https://w3-connections.ibm.com/wikis/home?lang=en-us#!/wiki/W96b34757c20f_4dc8_8415_b4904617cb62/page/Guided%20Evidence%20Management

Requirements

The requirement here is for a developer to be able to specify a page or pages in a wizard which should which are considered to be mandatory. The wizard will display an asterisk next to the page name in the wizard progress bar. When an asterisk appears in the progress bar a piece of text should be displayed in the modal title bar explaining which the asterisk indicates.

Design Options

These design options refer to including the mandatory indicator in the wizard progress bar.

Option 1 - Force developer to specify the mandatory pages in the wizard configuration file.

Winner! Not very invasive. Allows access to the information at the time when it is most relevant.

Option 2 - Use UIM syntax to enable a developer to specify a screen as mandatory.

This may not be the best approach because the progress bar is rendered separately from the UIM screen. There is currently no UIM syntax to specify this so new syntax would need to be introduced.

 

These design options refer to including the additional text in the modal title bar.

Option 1 - Inject HTML into parent title bar when building wizard progress bar.

Discounted: less performant than including the HTML by default in the widget HTML template.

Option 2 - Include "required" text in all title bars and only reveal when appropriate.

Winner!

Design

There really wasn't much design involved here. The decision was made to add the property to specify the mandatory pages to the properties file rather than the associated UIM page. This seems to be a more natural fit and is less invasive than adding introducing new UIM syntax to support it.

Implementation Notes

The implementation can be divided into 2 parts:

1. Adding the mandatory icon into the progress bar

2. Adding the explanation of the mandatory icon into the modal title.

1. Adding the mandatory icon into the progress bar

The progress bar is generated from a single renderer, WizardProgressBarRenderer. This renderer will be updated to inspect the new property, Wizard.Mandatory.Pages. As the renderer loops over the the pages it should check to see if a page is included in the list of mandatory pages and if it is the mandatory icon will be included immediately after the page title. The image used is the standard "mandatory" image. Accessiblilty should be a consideration.

 

2. Adding the explanation of the mandatory icon into the modal title.

 

This is a little trickier than than the mandatory icon. The modal title bar is constructed outside of the iFrame that hosts the wizard. For this reason the dojo publish/subscribe model will be used to display the required text. The modal template will be update to include the text, hidden, by default. This hidden text is then only displayed when appropriate. The progress bar renderer will include a piece of Javascript which listens for the "/curam/dialog/displayed" event. When this event is found it publishes a new "show/required/text" event. The listener for this event is in ModalDialog.js which removes the "hidden" class from the HTML element containing the text.

 

Application Impact

No impact. Wizards that do not specify the Wizard.Mandatory.Pages property should continue to work in exactly the same way as they did before.

Documentation/How To Use Feature


The feature will be used by developer when defining the Wizard. A new property is available to the developer to specify which page titles in the progress bar should include the mandatory '*'. When at least one page in the wizard is specified as mandatory, the modal window containing the wizard should include a configurable piece of text which will explain with the '*' mean e.g. "* Required".

The new property is named Wizard.Mandatory.Pages and it's value must be a comma separated list of integers e.g. Wizard.Mandatory.Pages=2,3

The "WIZARD_PROGRESS_BAR menu" section should be updated to include this new feature.
