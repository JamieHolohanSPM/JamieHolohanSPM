Curam TI CDEJ
Passing hidden values with an Editable List row

Design Page for support for hidden values to be passed as list row data

Background

This feature was required for Guided Evidence Management, specifically for their relationships page. That particular page was considered to be the "wow" for case workers who were shown the prototype so the feature was "must have".

Requirements

The requirement is to allow the passing of data hidden data, e.g. a concernRoleID, alongside data entered in an editable list.

  • 143955  - Editable lists need a way to uniquely identify a row upon submitting the page

Design Options

1. Allow CONNECT as a child of list:

<LIST>
.
.
.
<CONNECT>
 <SOURCE NAME="listAllCases" PROPERTY="concernRoleID"/>
 <TARGET NAME="EditCases3" PROPERTY="concernRoleID"/>
</CONNECT>
  </LIST> 

Pros:
- [Conor] Semantically this is what a developer would "expect".

Cons:
- Connect is not currently allowed as a child of UIM
- Connect is not currently treated as a "list" element so change is invasive.
- We need to implement the submission of the data
- [Conor] It's very late in the 6.2 release cycle to introduce such an unknown, far-reaching change.  Although maybe a development spike will prove it's not as fraught as it sounds.

 

2. Allow a FIELD to be tagged as "hidden"

<LIST>
.
.
.
    <FIELD HIDDEN="TRUE">
 <CONNECT>
<SOURCE NAME="listAllCases" PROPERTY="concernRoleID"/>
 </CONNECT>
 <CONNECT>
<TARGET NAME="EditCases2" PROPERTY="concernRoleID"/>
 </CONNECT>
    </FIELD>
</LIST>

Pros:
- Cheaper. Only change is to "hide" field (using CSS probably) and not "count" it when we are calculating field widths.
- Submission of data will be handled by existing infrastructure
- [Conor] A safer option to deliver this late in the 6.2 cycle. Less to go wrong.

Cons:
- [Conor] Semantically incorrect as it's not a field - another quirk to learn in UIM.

 

3. Introduce a new UIM element to support this requirement

Winner!

It was felt that introducing a new UIM element was the safest option. This isolates the changes from impacting on existing functionality. The new UIM element is to be name LIST_CONNECT
