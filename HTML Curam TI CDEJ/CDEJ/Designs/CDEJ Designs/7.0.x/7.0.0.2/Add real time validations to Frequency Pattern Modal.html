Background

When user enters wrong input like alpha numeric, values less than 1 and no values are selected for numeric field, the Progress Spinner Does Not Disappear. This happens with all the fields (Daily, Weekly, Monthly, Bi-monthly and Yearly fields) whenever incorrect values or no values are selected.
After Closing the modal 'Select a frequency Pattern' using 'x', spinner also seen on parent dialog , it doesn't stop and allow contents to get focus.

 

Requirements

List of Requirements

  • Requirement 1: The Daily input field of Frequency Editor the user can enter values between 1 and 99. This is the valid inputs therefore a validation error or progress spinner should not be displayed here. But if the user enters no value or values less than 1 in the daily input field, the input field should be highlighted red and the save button should be disabled.
  • Requirement 2: The Weekly input field works the same way as the Daily input field. The user can enter values between 1and 99. This is again the valid inputs therefore there will not be a validation error or progress spinner displayed. But if the user enters no value or values less than 1 in the daily input field, the input field should be highlighted red and the save button should be disabled.
  • Requirement 3: The Monthly section has four input fields and each fields has different validation error.

In the first input field, user can enter values between 1 and 99. If user enters no value or values less than 1 the input field should be highlighted red and the save button should be disabled.

In the second input field, user can enter values between 1 and 28 only, if user enters no value, values less than 1 and values greater than 28, the input field shall be highlighted red and the save button should be disabled.

In the third input field, user can input values between 1 and 99. If user enters no values or values less than 1 the input field should be highlighted red and the save button should be disabled.

  • Requirement 4: The Bi-monthly has two input fields, user can enters values between 1 and 28 only for both the fields, If user enters no value, values less 1 or values greater than 28, the input fields should be highlighted red and the save button should be disabled.
  • Requirement 5: The Yearly has a dropdown list for selecting months and only one input field, the input value depend on the month selected by the user. As each months has different numbers of days. If user selects January, March, May, July, August, October and December months from the dropdown the input values should be between 1 and 31. If user enters no value, values less 1 or values greater than 31, the input field should be highlighted red and the save button should be disabled.

           If user selects February month from the dropdown the input values should be between 1 and 28. If user enters no value, values less 1 or values greater than 28, the input field should be highlighted red and the save button should be disabled.

  If user selects June, April, November and September month from the dropdown the input values should be between 1 and 30. If user enters no value, values less 1 or values greater than 30, the input field should be highlighted red and the save button should be disabled.
