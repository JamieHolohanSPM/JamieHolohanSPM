Background

The "Add to my Tasks" and "Assign to" fields are presents on many pages of the application at different context levels.

 

Problem

The current markup for required fields * (required) used in Curam is not suitable in this situation as either Add to My Tasks or Assign To are required fields. The current state of the application does not provide a mechanism to indicate the user in advance that either fields are mandatory. It is only when the user tries to validate his inputs and that neither of the mandatory either fields are filled in that a warning message is being thrown. The first issue here is that the application is not informative and that causes an accessibility gap. As far as the voice over accessibility feature is concerned, there is no such feature to tell the user that either Add to my Tasks or Assign to fields are mandatory fields. This is the second issue that causes a further accessibility gap.

 

Requirements

The application should provide to any users and more importantly to users with disabilities enough information so that the user exactly knows where his inputs will be required before a validation check can be processed by the application.

 

List of Requirements

  • Requirement 1: Informative text available on the screen.
  • Requirement 2: Informative text being read by a screen-reader before the either one mandatory input fields.

Solution

1 Adding a DESCRIPTION attribute to the Assignement Details cluster populates the desired message on the .vim files.

<CLUSTER
    DESCRIPTION="Cluster.Title.AssignmentDescription"
    LABEL_WIDTH="22.5"
    TITLE="Cluster.Title.AssignmentDetails"
  >

The DESCRIPTION attribute value being defined in the corresponding .properties files.

2. Make sure that the text is backed-up with ARIA feature so that it can be read by screen-readers. Updating template in TI for Description atttribute (@DESCRIPTION) in gen-jsp.xsl:

 

<xsl:if test="@DESCRIPTION and not (@DESCRIPTION = '' )" >
             <xsl:attribute name="title"><xsl:value-of select="@DESCRIPTION"/></xsl:attribute>
             <xsl:attribute name="tabIndex">0</xsl:attribute>
</xsl:if>
