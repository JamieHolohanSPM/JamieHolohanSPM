Problem

As far as the voice over accessibility feature is concerned, there is no such feature to tell the user that either Case Participant, Participant or Name are mandatory fields. The current markup for required fields * (required) used in Curam is not suitable in this situation as either of these fields are required fields. The current state of the application does not provide a mechanism to indicate the user in advance that either fields are mandatory. It is only when the user tries to validate the missing inputs that a warning message is being thrown. The application is not informative and that causes an accessibility gap.

 

Requirements

The application should provide to any users and more importantly to users with disabilities enough information so that the user exactly knows where his inputs will be required before a validation check can be processed by the application.

 

List of Requirements

  • Requirement 1: Informative text available on the screen.
  • Requirement 2: Informative text being read by a screen-reader before the either one mandatory input fields.

Solution

  1. Adding a DESCRIPTION attribute to the Assignement Details cluster populates the desired message on the .vim files.

  <CLUSTER
     DESCRIPTION="Cluster.Title.ProvidedByDescription"
    LABEL_WIDTH="38.5"
    TITLE="Cluster.Title.ProvidedBy"
  >

 

The DESCRIPTION attribute value being defined in the corresponding .properties files.
