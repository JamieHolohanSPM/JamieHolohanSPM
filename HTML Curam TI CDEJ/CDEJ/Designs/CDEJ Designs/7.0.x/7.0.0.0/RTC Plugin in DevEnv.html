RTC Plugin in DevEnv

vNext Stack 1 delivers the RTC plugin into your Eclipse workspace, so there is no need to keep the RTC Client and development Eclipse open at the same time.

In order to enable tracking of changes in your workspace, the RTC sandbox must be registered with the plugin.

  1. Open Window -> Preferences, and navigate to Team -> Jazz Source Control -> Sandboxes
  2. Add the RTC workspace (e.g. C:\RTC-Dev\BizInf-vNext) as a Sandbox, and click OK to close the Preferences dialog.
  3. Open Window -> Open Perspective -> Work Items (go to Other > Work Items, if not already visible)
  4. Create a repository & project connection:
    1. Repository: https://jazz031.hursley.ibm.com:9443/ccm
  5. Select all of the projects in Project Explorer, right-click, and select Team > Share Project
    • Select Jazz Source Control as the repository type, and click Next
    • Click Finish
    • This will create the relevant mappings between projects in the Eclipse workspace and the RTC Sandbox.

For a better experience, in the Java / Java EE perspective, add two additional views:

  • Work Items > Work Items
  • Jazz Source Control > Pending Changes

 

If the Work Items view or perspective is not available, go to C:\CC\DevEnv\7.0\eclipse-extensions\rtc_plugin and remove artifact.jar and contents.jar
