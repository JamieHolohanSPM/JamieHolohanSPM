Background

In an expandable list, the Users see more information than is displayed in a simple list, without having to navigate away from the page that contains the list.

Each item from the expandable list allows the User to expand rows by clicking a toggle control.

When the User expands the item, a page that is relevant to the row is displayed. 

For each list that is expanded, a new iframe is created to display the content of the inline-page.

 

Problem

When the expandable list is collapsed and the new iframe is created to display the in-line page, the user can not the full page, some of the fields get truncated

The print functionality displayed the in-line page details from the expandable list gets truncated.

                      Screen: Edit Evidence page with expandable list opened                                                                                     Print Preview: Edit Evidence with expandable list opened missing the Comments

 

              

                                                                               

 

Support ticket: https://jazz031.hursley.ibm.com:9443/ccm/web/projects/SPM#action=com.ibm.team.workitem.viewQueries&tab=owned

 

 

Design

Here are the details of the solution:

1-The application verifies if the page has any expandable list open without context panel.
2-If so, The application recursively scans every iframe inside the expanded list.
3-For every iframe found, the application copies the contents of the iframe to a temporary wrapper (div) in the parent iframe.
4-Hides all the iframes inside the expanded lists.
5-At this point we have only 1 main iframe with all the contents of the expanded list.
6-Print the page
7-After the print, remove all the temporary wrappers created and show the expanded list iframes again (back to the original state).

To remove impacts, this solution is only executed on pages with expandable lists with at least 1 list expanded without context panel.

 

Notes: 

  1. Design assigned only for the expandable list without context panel inside.
  2. The context panel contains the CSS layout across different streams. The timeline to fix the support ticket didn't allow us to work on it in this version. However, it can be introduced in the next version
  3. The design solution built can be expanded for more scenarios in the application that involves multiple iframes. 

 

Implementation Notes:

 

CDEJ:

  • Util.js

    • _printMainAreaWindow(window) :  

      1. Check if the window contains an expandable list expanded

        1. Yes, execute the method _prepareContentPrint to copy the iframe content inside the wrappers(div), print the content and revert the page to the preview state

        2. No, set the focus and print the content

    • _prepareContentPrint(window)

      1. Recursively, prepare content to copy the body of iframe inside a wrapper and disable the iframe when the user is printing the page.

    • _deletePrintVersion

      1. Remove all the temporary wrappers created and show the expanded list iframes again (back to the original state).
         

  • curam_lists.css, v6_common.css
    • Styling the wrapper in the print style.
