Problem 

JAWS does not indicate the state of the sorted column.

For example, On the “Register person” JAWS read that column are sortable but when sorting, the state of the sorted button doesn’t get communicated to JAWS users.

Also, for user experience, it’s better to also visually show the sorting buttons on the table, so that visual users would know the table is sortable. (not implemented yet)

Steps to reproduce

  1. Launch Register person and search for Last Name "Smith"
  2. The Search Results will return all users with the same last name "Smith"
  3. Click on any column, e.g Person, or Address


Actual result
JAWS read 'Person sortable column link' or 'Address sortable column link' without letting JAWS users know that the column has been sorted, either ascending or descending

Expected Result
JAWS should indicate the column has been sorted either ascending or descending
