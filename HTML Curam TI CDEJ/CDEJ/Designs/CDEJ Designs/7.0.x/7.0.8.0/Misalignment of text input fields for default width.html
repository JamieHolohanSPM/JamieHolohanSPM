Background

On create/edit screens in the application, the text input field is not aligned correctly with other form controls when it is being displayed using it's default width i.e the WIDTH attribute is not set on that field in UIM/VIM. It should align with other fields on a screen that has other form controls. It seems that this was a defect for V7.

 

The source of the issue is that 20 pixles are subtracted from the width of 100%. The following CSS selectors control the layout of the text input field in a cluster (curam_form_controls_text_input.css):

body.soria div.list table tbody td.field input.text,
body.soria div.cluster table tbody td.field input.text,
body.soria div.cluster table tbody td.field input.password,
body.soria div.cluster table tbody td.address input.text,
body.soria div.cluster table tbody td.batch input.text {
 width: calc(100% - 20px);
}

Requirements

When there is no WIDTH specified on a text FIELD (the fields domain is based on CURAM_STRING) element in UIM/VIM) it should be aligned with other form controls on a create/edit screen in the application.

Design Options

Option 1

             On the text input field, remove the 20 pixels so that is juts left at 100%

Option 2

Leave the text input field as is but instead update the CSS for the other form control e.g for date, dropdown fields, subtract an additional 20 pixels so that they are all aligned.
