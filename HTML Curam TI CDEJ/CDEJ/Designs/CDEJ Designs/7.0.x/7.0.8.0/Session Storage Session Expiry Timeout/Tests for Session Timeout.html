Tests for Session Timeout

Testing

Manual Testing

Setup configuration before running any of tests below

• Add and publish the following properties as sysadmin:

• External application configuration :
- curam.environment.enable.timeout.warning.modal = true

- curam.environment.timeout.warning.modal.time = 120


• Internal application configuration :
- curam.environment.internal.enable.timeout.warning.modal = true

- curam.environment.internal.timeout.warning.modal.time = 120

 

For any tests that needs any of above values changed we mention in Individual tests


• On WebSphere console: >>> https://servername:9043/ibm/console   Login: websphere/websphere
1. Servers -> Server Types -> WebSphere Application Servers
2. CuramServer
3. Container Settings -> Session management
4. Session Timeout -> 3 minutes
5. Apply; Save;

6. Go back to Session management

7. Click on Enable Cookies link

8. Check both Restrict cookies to HTTPS sessions and Set session cookies to HTTPOnly to help prevent cross-site scripting attacks

9. Apply; Save;

10. Logout
11. Stop and start the application server

 

• Plan 1 - Verify that Session Timeout Warning modal appears in both Internal & External Application when it's enabled & Session time out Warning time approaches && functions (Continue, Quit, Timeout) of Session Timeout Warning Modal works as expected


• In the external application: for this test consider the timeout element configured as : <ac:timeout-warning timeout="120"/>
• Create a User Account in the external application & login.
• After 1 minute of idle idle time, the session timeout warning dialog should appear with the Continue button and Quit button and a 2- minutes timer.
     - If the user clicks on the Continue button:
            1) Close the modal dialog
             2) Resets the session
     - If the user clicks on the Quit button:
            1) Close the modal dialog
            2) The application will redirect to the logon page.

    - If the timer counts down to 00:00, the application will redirect to the logon page, hence invalidating the session with the message " You have been automatically timed out due to a period of inactivity on your account.".

• In the Internal application: for this test consider the timeout element configured as : <ac:timeout-warning timeout="60"/>
• Login as caseworker in the internal application
• After 2 minutes of idle time, the session timeout warning dialog should appear with the Continue button and Quit button and a 1- minutes timer.
     - If the user clicks on the Continue button:
            1) Close the modal dialog
            2)  Reset the session
     - If the user clicks on the Quit button:
            1) Close the modal dialog
            2) The application will redirect to the logon page.
    - If the timer counts down to 00:00, the application will redirect to the logon page, hence invalidating the session with the message " You have been automatically timed out due to a period of inactivity on your account.".
