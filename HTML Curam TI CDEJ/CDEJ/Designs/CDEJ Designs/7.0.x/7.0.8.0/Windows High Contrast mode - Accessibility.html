Windows High Contrast mode - Accessibility

 

Design Page for Windows High Contrast mode missing icons

 

Background

The background to this wiki page is Support Ticket 239933 which outlines the issue that currently OOTB with the high contrast mode turned on in Windows OS, there are multiple non-decorative icons and images missing. Currently, we are waiting on feedback from the customer on the scope of this ticket as general issues are mentioned but the screenshots only contain some defects.

 

The first defect around this issue (High Contrast mode) resulted in a meeting held on 4th March 2019 (with Fergal Gavin, Barry O'Sullivan, Francesco Marini, Shane McFadden and Clara Mannion) to discuss where we stand on the accessibility issues arising from Windows high contrast mode turned on. The conclusion from this meeting is that IBM requires any teams to include Windows HC mode in their testing steps and we should fix any defects in this area.

Technical Analysis

Windows HC mode: ON

Missing icons highlighted on Person Home page

 

1. Missing arrow icon on top right of Person image (Citizen Context Viewer icon)

This contains a link <a> tag with a background url to display the image and css changes to update the background image on hover and focus states.
The link <a> tag already has an existing image tag, that is currently hiding through css styling (display: none).

Solution:
- update PersonDetailsPanel.css to remove background images and view the <img> tag.
- update LinkBuilder.java to include an attribute on the link, so when in focus the image will be updated.

- update Image.properties to add hover icon image.

- update ImagePanelBuilder.java

 

2. Missing phone, map and email icons

This are also background images and an image tag needs to be added.

- update PersonDetailsPanel.css
 

3. Missing toggle buttons on Home tab page.

This icon is a background image on a button tag.

Update the toggle button to have <img> tag.
 

4. Missing menu icon on top right of Person home page.

This icon is in a span, with the background image on the dijitButtonNode class which is updated through css on a hover event.
 

5. Shortcuts icon 

This missing icon is of a right and left double chevron images, depending on whether the Shortcuts panel is expanded or not. The images are background urls to the <div> tag.

Solution:

- Update ExpandoPane.html to have <img> tag

- Update ExpandoPane.js
 

Note:
1. Also update the css files corresponding to right to left variations and test from right to left.

2. Test with JAWS screenreader on what's in focus, and what's read.

Common issues

 

Resulting from the technical analysis and investigation into this support ticket, there are some common issues and solutions. The reason theses icons are missing are because Windows HC mode removes all background images. Instead, they need to be updated to have an image tag. 

Example:
<img src="smiley.gif" alt="" >

The image tag must have the alt attribute, as it is a HTML standard. However, depending on whether they are decorative images or not, it will depend whether the alt attribute should be null (alt="") or have appropriate alternative text. Refer to here: https://webaim.org/techniques/alttext/

 

Another issue is that generally the background images have been used for icons, as it's very easy to update the images through css when the state of the element changes, such as a hover or focus. 

For an image tag there are two ways to implement this behaviour. 
1. Add javascript function to the link, so that when it's on hover or focus (this can be a large range of mouseover, mouseout, keyboard actions etc) to update the image src.
2. Or add two <img> tags in the link, and hide one. And using css display the correct image for that state.
