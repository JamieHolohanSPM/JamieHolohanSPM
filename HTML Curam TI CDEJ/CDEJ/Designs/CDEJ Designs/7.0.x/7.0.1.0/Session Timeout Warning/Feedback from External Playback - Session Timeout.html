Feedback from External Playback - Session Timeout

Date : 23rd November 2016

 

Action points following feedback from relevant stakeholders

 

  1. Shane to check previously received feedback from NYC based on the initial delivery of session timeout warning for External to ensure we have addressed the customers needs with this second implementation. Feedback from Graham following the NYC PC Bank visit in 2015 was that when citizens use a shared PC in the PC Banks, a new user can often sit down to a message saying "your session has expired". This is confusing for the citizen as they have only just arrived in the PC Bank and not yet created a session.
  2. The message on the homepage "your session has expired" should only be displayed to users who have performed an action on the page (e.g clicked the 'Quit' button). Bruna to ensure that this is the implemented functionality.
  3. Customers tend not to use our login pages, we need to ensure that the process to implement this functionality on custom login pages is documented accordingly. Bruna to ensure that the relevant documentation has been completed for customers to update their own login pages.
  4. This functionality is based on the session timeout value set on the app server, we should ensure it's tested on all supported servers and not only WAS. Bruna to test on all other relevant servers.
  5. The 'Quit' text on the button displayed in the popup could be confusing to customers. It could lead the customer to believe that clicking 'Quit' will cause only the popup to disappear. Shane and Sara to review whether the text on the 'Quit' button is sufficient.
  6. Although we know there is a homepage delivered in UA, we need to check whether there is any other homepages delivered in the application that we need to update for the release.
