This enhancement is designed to improve the usability of the application. The current  design limits the available visible space to display list results and forces the user to scroll down to see search results.

If the new enhancement contains any User Interface element, then a UX spec may be required. The spec should be associated with this design.

Note: New UI features should also be tested to ensure they work on a tablet device. From 7.0.1.0 we state we support tablet devices so we need to make sure we are not regressing in this area.

Key board short cuts should also be considered if developing features that interact with the user depending on mouse clicks/key board interaction (i.e. back browser button, refresh etc.)

Browser differences should be considered for features being developed ensuring that the required browsers are tested and supported.

Is the feature specified for the internal caseworker application or for the external Universal Access application (or both)..

Accessibility for both the desktop application (JAWS and IE11 is the supported combination) and tablet (VoiceOver on an iPAD using the keyboard and gestures is the supported combination) should be considered for new UI features.

Table Support

 

  • As of v7.0.1 Curam will be certified for use on tablet devices. This introduces a new responsibility on Business Analysts, UX Designers and Developers to ensure that they do not change existing or introduce new features that break the Tablet support statement. For the most part this is limited to the production of custom widgets (non-standard UIM) and how they perform when using touch screen controls. Please note the following:
  • Tablet resolution is typically lower than the minimum recommended resolution of 1366x768. This means that the rules around wrapping of text have been relaxed for tablet, but teams should make a reasonable effort to ensure that any wrapping is not excessive and does not affect the ability to understand and use the application.
  • Where custom features are introduced, teams must test using a iOS and Android device to ensure that the feature is compatible with touch screen controls.
  • Where custom features are introduced, they must meet accessibility requirements for mobile devices.

Implementation Notes

The Tool to Identify, list and remove the UIMs-VIMs containing fixed height scroll-able elements is detailed in task 227293

https://jazz031.hursley.ibm.com:9443/ccm/web/projects/SPM#action=com.ibm.team.workitem.viewWorkItem&id=227293

 

The source controlling of the tool is tracked by the task 232197

https://jazz031.hursley.ibm.com:9443/ccm/web/projects/SPM#action=com.ibm.team.workitem.viewWorkItem&id=232197

The tool has been integrated in TI/test/UiLayoutHelper folder.

Part of this tool was already there, we overwrite the old version with the current one.

 

The tool offers the ability to :

  1. Identify and list the UIMs containing these fixed height elements.  - Done
  2. Generate a summary log file that lists the height set in pixels, the UIM/VIM name, and to list the url to the CADG page showing the page preview for that UIM/vim   - Done
  3. Have a "Clean up " mode that will remove the elements identified in (1)   - Done
  4. Generate a consolidated CADG .html page. This required the tool to generate an output file as per the POC work tracked in Task 226042 https://jazz031.hursley.ibm.com:9443/ccm/web/projects/SCM%20Support%20Hub#action=com.ibm.team.workitem.viewWorkItem&id=226042   - Done
  5. Add a black / white list functionality to the tool This will allow a user to specify by means of a csv file a list of UIMs which the tool will not to "clean" - Not Implemented

- This will support the possible case where test or business identify pages that should not be changed proposal that if the list is empty (or does not exist) all UIMs will be "cleaned"
 

This tool will be used in conjunction with a new DevOps Pipeline that is described in details in Enhancement 226690

https://jazz031.hursley.ibm.com:9443/ccm/web/projects/SCM%20Support%20Hub#action=com.ibm.team.workitem.viewWorkItem&id=226690

The new pipeline offers the ability to :

  1. Deploy the Curam application, for example a platform or cross solutions build
  2. To make wide spread UI updates to the application  for example 500+ UIM/VIM changes on a cross solutions build 
  3. To generate before and after consolidated preview pages displaying a large number of the actual UI screens for easy comparison
  4. To give test the ability to test a fully functional application
  5. and all without changing any of the checked in code
