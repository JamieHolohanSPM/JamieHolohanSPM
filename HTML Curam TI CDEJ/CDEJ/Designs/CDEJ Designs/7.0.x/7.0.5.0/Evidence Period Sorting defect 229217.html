Background

Three APIs were delivered for MSD to resolve issues they were having.  The results for those APIs "[...] will be ordered in descending order by the business end date of the evidence (if available), with open (no end date) entries returned first. " (IBM Cúram Social Program Management 6.0.5.8 iFix1, IBM Cúram Social Program Management 6.1.0.1 release notes)http://www-01.ibm.com/support/docview.wss?uid=swg27045661 ; http://www-01.ibm.com/support/docview.wss?uid=swg27046293

 

During testing the requirement to handle future start dated evidence was also raised.

Requirements

During discussion with business it was decided the API's should return Evidences in the following order of precedence:

  1. Start date in the past, no end date.
  2. Start date in the past, end date in the future.
  3. Start date in the future, with latest end date.
  4. Start date in the future, with earlier end date.
  5. Start date in past, end date in the past.

Architecture/Design Decisions

 

Architecture Decision Title Issue or Problem Statement Assumptions Status
Update the evidence period comparator to evaulate zero as well as null dated periods. The Evidence Period comparator originally handled null, but not zero dated evidence periods. This meant that if an end date was set to Curam default for null it was not correctly evaluated as being open The evidence period comparator may receive periods with null end dates, as well as zero end dates, these should be evaluated as the same thing. Closed.
Update the evidence period comparator to evaluate future start dated Evidence Periods correctly. The Evidence Period comparator originally contained logic to check for end date differences between the Periods, and return a precedence based on that. The comparator should also handle future dated evidence, which is not yet current. The case worked wants to see current evidnence displayed at the top of the list, future dated is not currently active, and so should be a lower precedence than current but higher precedence than closed. Closed.
