Design Page for Improve Printing in the Curam Product

Background

There are a number of printing issues in the application. Printing in Curam is highly coupled with the browser CSS. This means when we bring in a new browser, browser update or we make a change. We run the risk of their being a browser issue. What appears to be a minor update, the impact of a change may be sever and lead to blank pages.

Testing can also be difficult. 

We have technical limitations because we have to render a number of iFrames.

Requirements

1)  A user needs to be able to print content in the content panel and have the print out in a PDF.

2)  All content in the content panel needs to be visible and printable in PDF form.

Q&A

  • Should the printing PDF functionality implemented for every pages on the application(e.g. drop down lists) or just some pages which costumers might want to print?
  • Should the printing PDF supports all the browsers(IE, FF, Chrome)?

Implementation Notes

Below is an example of java code used to convert the HTML document to PDF.

import java.io.*;
import com.lowagie.text.DocumentException;
import org.xhtmlrenderer.pdf.ITextRenderer;

public class FirstDoc {
    
    public static void main(String[] args) 
            throws IOException, DocumentException {
        String File_To_Convert = "test.htm";
        String url = new File(File_To_Convert).toURI().toURL().toString();
        System.out.println(""+url);
        String HTML_TO_PDF = "ConvertedFile.pdf";
        OutputStream os = new FileOutputStream(HTML_TO_PDF);       
        ITextRenderer renderer = new ITextRenderer();
        renderer.setDocument(url);      
        renderer.layout();
        renderer.createPDF(os);        
        os.close();
    }
}
