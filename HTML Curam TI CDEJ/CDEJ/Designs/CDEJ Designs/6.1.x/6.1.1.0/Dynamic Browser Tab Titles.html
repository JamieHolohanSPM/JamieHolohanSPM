Curam TI CDEJ
Dynamic Browser Tab Titles

Page titles in SPM are missing meaningful page titles 1.2.4d

Subtitle: "Dynamic Browser Tab Titles" for short

 

Background

Page title in SPM is currently hardcoded to "IBM Curam Social Program Management". This is not meaningful for a screen reader user and therefore fails VPAT checkpoint 1.2.4d.

Page title should include context (what section is open, application tab is open, context panel name, content panel name). This information will need to be updated using JavaScript as the user navigates through the application.

 

Requirements

The browser tab title should update to include a meaningful title for the visible page. E.g. "My Payments - IBM Curam Universal Access" or "Person Search - IBM Curam Social Program Management".

  • TEC-18049 - Page titles in SPM are missing meaningful page titles 1.2.4d (TI changes)
  • TEC-18886 - Page titles in SPM are missing meaningful page titles 1.2.4d (IEG changes)
  • GUM-6153 - Page titles in SPM are missing meaningful page titles 1.2.4d (UA changes)

Implementation Notes

The most important parts are two functions in TI file util.js:

curam.util._setupBrowserTabTitle(staticApplicationTitle, separator, displayApplicationNameFirst)

This function is called by either AppController.jspx or ExternalApplicationBannerRenderer.java depending on whether user is using the internal or external application. The method saves the given properties for later use.

  Internal App External App
When is function called? AppController.jspx ExternalApplicationBannerRenderer.java
Params come from CDEJResource.properties CITWSAPP.properties (or other similar .app file)
Param for application title string browser.tab.title landing.title
Param for separator string browser.tab.title.separator browser.tab.title.separator
Param for application title first boolean browser.tab.title.application.title.first browser.tab.title.application.title.first

See the javadoc in util.js for more info.

curam.util.setBrowserTabTitle(/*optional*/ name)

If given a name value, this function sets the browser tab title to "<name><separator><application title>" or "<application title><separator><name>", depending on the value of "browser.tab.title.application.title.first", e.g. "Select Screening - IBM Curam Universal Access".

 

In the external application, curam.util.setBrowserTabTitle is called by the following files:

  • PagePlayerRenderer.java
  • LandingPageRenderer.java
  • UIMIFrameWrapper.java

The following files call RendererUtils.setPopupWindowPageTitle as due to them being pop-up windows they do not have access to the application.jspx browser context and thus are unable to get the settings from CDEJResource.properties. They instead use properties from ProviderServiceModal.properties or else fall back to default values:q

  • GetDirectionsRenderer.java
  • ProviderModalDetailsMapContainerRenderer.java
  • ReferralFieldRenderer.java

 

In the internal application, curam.util.setBrowserTabTitle is called by the following functions:

  • curam.ModalDialog._loadErrorHandler

  • curam.ModalDialog._modalClosedHandler

  • curam.UIMController.processFrameLoadEvent

  • curam.tab.TabSessionManager.tabSelected

  • curam.tab.TabSessionManager.tabSectionSelected

  • curam.ui.UIController.revealTabNow

  • curam.util.TabNavigation.postIframeLoadHandler

In IEG, curam.util.setBrowserTabTitle is called by the following files:

  • ieg-bootstrap.js
  • update-manager.js

In the external application, curam.util.setBrowserTabTitle is called by the following functions:

  • curam.widget.containers.TransitionContainer._setDisplayPanelAttr

 

Typically in the internal application it will be called without a name parameter. If not given a value for 'name' the method will call curam.util._findAppropriateDynamicTitle() which attempts to find an appropriate title in the DOM. It searches for:

  • Page Title on Content Panel
  • If no page title on CPanel, then get title from page navigation group,
  • If no page navigation group title then use title from navigation tab bar
  • If no title on navigation tab bar then use title on application tab
  • If no title on application tab then use title on Section.

If none of the above are found it will default back to displaying the Application Title on its own. It uses the first item found.

It also searches for a sixth item, for use with UIM pages in external application (for when curam.ModalDialog._modalClosedHandler gets called in that context.) This is because when closing a modal dialog window we need to trigger the check to update the title to the underlying page.
