Appeals Clause Picker

Design Page for File Edit Search page functionality (a.k.a. Appeal clause picker)

 

Background

This feature was available prior to 6.0. We are reintroducing it specifically for California DIR who are threatening legal action if the feature is not available in 6.1.

Requirements

The requirement is as follows:

- It should be possible to configure a search page as a FILE EDIT widget parameter. The specified search page should be a UIM page.

- When the widget is launched the user should have access to the configured search page.

- If the search page contains a CLIPBOARD element, clicking the clipboard link should insert the copied text into the open Word document at the current cursor position.

Option 2

Pass the text to be inserted into the document directly to the document via Javascript & the applet rather than replying on the system clipboard.

Design

UI:

A "search" button has been added to the new FILE_EDIT control modal. Clicking this modal launches the search page, in a modal, on top of the control modal. The search page includes an action link which triggers the insertion of the text into the document.

 

Functional:

When the action link is click a javascript function is executed. This function executes a DOJO publish event, passing the text to be inserted, to the parent window. The parent window has subscribed to this event and executes a new operation on the applet. The implementation of this applet function executes three JACOB calls to inset the text into the document:

1. Get the current cursor position

2. Insert the text

3. Press the "arrow" key so that the text is not highlight (this prevents the user inadvertently overwriting the inserted text.
