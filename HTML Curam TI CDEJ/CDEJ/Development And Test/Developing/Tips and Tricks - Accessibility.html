IOs - Debugging JS code or CSS

Sometimes it is hard to identify problems in the web page running in IOs devices. The safari browser provides a way to connect the IOs device with Safari on Mac OS x.

Following the steps to enable it:

1) Enable developers tools in Safari

2) Enable Web inspector in IOs 

    - Select Safari in the Settings app and tap the Advanced option. Then turn on the Web Inspector.

3) Connect the IOs device with the Mac Os with a USB cable.

4) Connect and debug

   - Open the Safari browser on both devices.

   -  In the Develop Menu in your Mac Os, you’ll see that your iPad now appears.

For more info: https://mattvanderpol.com/2013/08/14/how-to-debug-javascript-or-css-on-ios/

Page focus depending on the rotor selected

voiceover allows the user to change the way how the voiceover navigates to the page.  

There is a list of option that allows the user to navigate a quick way in all the buttons, list, tables, containers and etc. 

These options may interfere when the page is opened. For example:

  1.  If the rotor is a text field when the page is opened the focus will be in the first text field that can receive focus
  2. If the rotor is a table when the page is opened the focus will be in the first  table that can receive focus

Focus not aligned with the field with voiceover on

There are scenarios in Cúram application where the focus is not aligned with the field when the voiceover set the focus there.

In some cases, we could identify that a css hidden is applied to the field(it usually happened with span).

Example problem:

 

In this case, the voiceover is reading the calendar title. However, the focus is not aligned with the image.

The HTML and CSS:

<span class="hidden">Click here to select a Date</span>

body.soria span.hidden, body.soria table .hidden, body.soria .dijitDialog .closeText, body.soria #app-banner-dc .hidden, body.soria div.access.hidden {
 position: absolute;
 left: -1000000px;
 display: inline;
}

To fix this issue, we have applied the css below for mobile devices:

 

body.curam.soria.mobile table.date td.input-date + td.popup-actions .hidden {

    position: absolute;

    left: 0px;

    top: 0px;

    right: 0;

    border: 0;

    clip: rect(0, 0, 0, 0);

    height: 0px;

    width: 0 px;

    margin: 0px;

    overflow: hidden;

    padding: 0;

    bottom: 0px;

    line-height: 1px !important;

}

 

Span tag HTML with role=button does not execute onclick function with Voiceover

 

A common scenario in the Cúram application is to create a button with the span HTML and add the role="button". However, it has been causing defects with voiceover. Voiceover does not execute the onclick function when this scenario is found. To fix this scenario change from span HTML to button, like this:

Before:

  <span aria-expanded="false" aria-label="3 column clusterToggle Button"

           onkeypress="if(curam.util.enterKeyPress(event)){toggleCluster(this,arguments[0]);}"

           onclick="toggleCluster(this,arguments[0]);" role="button" tabindex="0" title="Toggle"/>

Now:

<button aria-expanded="false" aria-label="3 column clusterToggle Button" 

           onkeypress="if(curam.util.enterKeyPress(event)){toggleCluster(this,arguments[0]);}" 

           onclick="toggleCluster(this,arguments[0]);" role="button" tabindex="0" title="Toggle"/>
