Set Up the Development Environment

Note: Setup is for Windows only - Mac development not supported on these legacy streams. 

 

  1. Run "install-appserver.bat wls"
  2. Create a file called setenv.bat containing the following commands:
    • set DEVENV_HOME=%CD%\devenv
    • set WLS_HOME=%CD%\devenv\tools\weblogic\weblogic92\server
    • set JAVA_HOME=%CD%\devenv\tools\weblogic\jdk150_04
    • set J2EE_JAR=%CD%\devenv\tools\weblogic\weblogic92\server\lib\weblogic.jar
    • set PATH=%CD%\devenv\tools\weblogic\jdk150_04\bin;%PATH%
    • set CURAMSDEJ=%CD%\release\CuramSDEJ
    • set CURAMCDEJ=%CD%\release\CuramCDEJ
  3. Add Bootstrap.properties and AppServer.properties files to TI\coreinf\properties folder.
    • Example files for you to edit and use:
      • AppServer.properties
      • Bootstrap.properties
  4. Run the following commands from root folder, "TI":
    • setenv.bat​​
    • anthill-runbuild.bat server client database clientmockapp 
    • anthill-runbuild.bat -f client/components/build.xml eclipse
    • rename .project .project.backup
  5. Start Eclipse:
    • setenv.bat
    • cd devenv
    • eclipse.bat
  6. In Eclipse:
    • Create a Repository Connection, https://jazz031.hursley.ibm.com:9443/ccm/
    • Click File > Import > Preferences > Select "<TI workspace root>\devenv\eclipseenv\curam.epf"
    • Change to the "Java" perspective
    • Click File > Import > General > Existing Projects into Workspace
    • Select the root 'TI' folder of your workspace and click Ok
    • Wait for Eclipse to finish searching for projects
    • Select the following projects and click Ok:
      • AppInf
      • client-persistence
      • ClientMockApp
      • CoreInf
      • general-plug-ins
      • Generator
      • JDECommons
      • plug-in-framework
      • renderer-plug-ins
      • ServerCore
      • test-framework
  7. Right-click on ClientMockApp > Tomcat > Remove Context Definition, then right-click on ClientMockApp > Tomcat > Update Context Definition
  8. Start Tomcat and open http://localhost:9080/ClientMockApp in your browser

 

 

Other Notes

In case you run into errors...

  1. Click Window > Preferences
    • Java > Compiler 
      • Set compiler to 1.5
    • Java > Installed JREs
      • Set jdk path to java 1.5 file location
    • Tomcat
      • Set version to 5.x
      • Set Tomcat home to devenv/tools/tomcat
  2. If you see errors about unbound classpath variables, e.g. "Unbound Classpath Variable: CURAMSDEJ/lib/jde-commons.jar":
    • Right-click on the affected project (e.g. ServerCore) > Properties > Java Build Path > Libraries
    • Take note of the path of the missing jar (e.g. CURAMSDEJ/lib/jde-commons.jar)
    • Remove the missing library
    • Click 'Add External JARs...' and select the required jar. For example, for a missing "CURAMSDEJ/lib/jde-commons.jar" select "<TI workspace root>\release\CuramSDEJ\lib\jde-commons.jar"
    • Do the same for any other missing jars
  3. If you have an error for Workflow_display_TH.java being unable to find curam.omega3.mockobjects.MockLog:
    • Right-click on ClientMockApp > Properties > Java Build Path > Libraries > Add External JARs
    • Select <TI workspace root>\release\CuramCDEJ\lib\curam\jar\omega3-coreInf.jar
