Running within CDEJ

SPM 6.1

For 6.1 we are using Intern, Selenium and Node JS to run units and report code coverage. On previous versions of the the Curam JDE (pre 6.1), Doh and JSCoverage were used.

 

Development environment

To run client-inf/mobile-sdk project tests do the following:

  1. In eclipse go to the External Tool menu/External Tools Configuration.

 

       2.  To debug or run tests, select the applicable configuration and click on the 'Run' button. It should be noted that if an external tools configuration has been executed once, then it will be available in the Menu.

NOTE 1: Before you run tests you must start the selenium-server. You can start the selenium server by running 'build -f build_scripts\client.xml selenium-start' from the command line. DEVENV_HOME should be set before doing so

NOTE 2:  To debug tests you must have chrome installed. To run tests you must have a supported version of Internet Explorer installed.

NOTE 3: To run selenium on IE, you must configure IE to have Protected Mode on all zones. See https://code.google.com/p/selenium/issues/detail?id=1795.

NOTE 4: To run selenium on IE9, you must open developer tools (F12) and set the properties Browser Mode to IE9 and Document Mode to IE 9 Standards. Also make sure that the compatibility mode is disabled (Tools -> Compatibility View Settings -> Uncheck all the three checks and remove all the addresses added to the list.)

Also more important see: https://code.google.com/p/selenium/wiki/DesiredCapabilities
