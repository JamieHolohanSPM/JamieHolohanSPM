Uitoolkit to run unit tests

Install Uitoolkit to run javascript unit tests

By Eclipse

  1. From the uitoolkit folder (/TI/reltools/uitoolkit) run:

                 PATH_TO_DEVENV/tools/nodejs-32bit/bin/npm install 

  1. Go to retools/uitookit/node_modules/selenium-standalone/bin/ and run:

                  PATH_TO_DEVENV/tools/nodejs-32bit/bin/node selenium-standalone install 

  1. Go to PATH_TO_STREAM/TI/reltools/uitoolkit/tools/intern/templates

                 File: debug-intern-tests
                 Update the paths

                 Example:

             <?xml version="1.0" encoding="UTF-8" standalone="no"?>
             <launchConfiguration type="org.eclipse.ui.externaltools.ProgramLaunchConfigurationType">
                  <stringAttribute key="org.eclipse.ui.externaltools.ATTR_LOCATION" value="/Users/CianMcCann/DevEnv/TI7030/tools/nodejs-32bit/bin/node.exe"/>
                  <stringAttribute key="org.eclipse.ui.externaltools.ATTR_TOOL_ARGUMENTS" value="node_modules/gulp/bin/gulp.js debug-intern-tests --webRoot /Users/CianMcCann/Streams/TI7030/TI/client/components/client-inf"/>
                  <stringAttribute key="org.eclipse.ui.externaltools.ATTR_WORKING_DIRECTORY" value="/Users/CianMcCann/Streams/TI7030/TI/reltools/uitoolkit"/>
             </launchConfiguration>

        

             (  Note: in using a Mac, do not type .exe (Windows file) 
               Example: <stringAttribute key="org.eclipse.ui.externaltools.ATTR_LOCATION" value="/Users/CianMcCann/DevEnv/TI7030/tools/nodejs-32bit/bin/node"/> )
 

  1. Put it in eclipse/client-inf folder and run it (This file can actually be put into any folder).
  2. In eclipse, locate the file, right click, Run As, debug-intern-tests.
  3. Once "Starting 'debug-intern-tests'..." appears in the eclipse console, Open: http://localhost:3000/node_modules/intern/client.html?config=intern-config/intern-config.js
  4. The JS unit tests should be run against all supported browsers!

 

The steps 1 to 4 only needs to be executed once.

 

By command line

  1. From the uitoolkit folder (TI/reltools/uitoolkit) run:

                 PATH_TO_DEVENV/tools/nodejs-32bit/bin/npm install 

  1. Go to TI/reltools/uitookit/node_modules/selenium-standalone/bin/ and run:

                  PATH_TO_DEVENV/tools/nodejs-32bit/bin/node selenium-standalone install 

  1. Go to TI folder and run:      ​

       ./appbuild test-component-javascript -Dcomponent=client-inf as this build fails when one or more js tests fail. 
  




Check javascript code coverage

Using the command line to run the tests first as it builds the coverage report (be sure to follow the first two steps to install selenium-standalone).

. You can see the code coverage in: TI⁩ ▸ ⁨client⁩ ▸ ⁨components⁩ ▸ ⁨client-inf⁩ ▸ ⁨html-report⁩ ▸ index.html

Note: to speedup your tests development, you can comment all the tests from the test-suite.js and leave just yours.
