Steps to review a ticket

 

  1. Design Page

    Check if the ticket is linked with any design page. Try to understand the problem and the solution. 

             - Check for diagrams that explain the problem and the solution.
             - Check if the uses cases are presented covering all the possible scenarios. Also, the testing tab in RTC is covering all the uses cases described in the design page.
     

  2. Release Notes

    Check if the ticket is required to have the releases notes(internal and external) and how the developer explained it. 
        - Wiki page release notes.

 

  1. Accept the code(wiki)

 

  1. Check the code
           - Check the code items reviews, wiki
           - Important: Check if all the pieces of code are tested by JUnit(for Java class) or Intern framework(for Java class).
           - The only code that it is impossible to test by the Unit test can be passed without the automatic test.
     
  2. Execute the test plan
     Go to the testing tab in RTC and execute the test plan.
      It is important the testing is covering all the scenarios. As well, test all the browser support by the application. See here the pre requisites

 

  1. Approve or reject(wiki)
