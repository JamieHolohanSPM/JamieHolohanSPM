Overview

If you are looking for the dojo upgrade instruction, please refer to Dojo Upgrade Instructions.

 

This page documents the specific information about IDX Upgrade in Curam.

At the highest level the required activities are:

  1.  Create a new ibm idx source ZIP for the target version, put on TINY & GIGANTIC and update installer to use the new ZIP.
  2.  Review and reapply Dojo overrides as appropriate.
  3.  Review and migrate (if needed) custom widgets that build on Dijit.
  4.  Test.

Each of these is discussed in more detail below. 

Create new source ZIP

The version of IBM Dojo Extension Toolkit (IDX , now it is re-branded to IBM Front End Toolkit from version 1.5) that you'd like to upgrade to can be downloaded from the IBM IDX Website, which will redirect you to the the iRAM (Rational Asset Manager). Note that you need to request access to this system in order to be able to download the IDX toolkit ZIP.

  1. Once the toolkit is downloaded. Extract the ZIP into a temporary directory.
  2. In the temporary directory, find the "idx-v1.x.x.x-x-YYYYMMDD-ibmproducts.zip" and extract it as well.
  3. Go to the directory where "idx-v1.x.x.x-x-YYYYMMDD-ibmproducts.zip" is extracted. Find the "idx-v1.x.x.x-YYYYMMDD.zip" and rename it to ibm_idx_source_<<version>>.zip (e.g. ibm_idx_source_1.4.2.6.ga.zip).  Note: you don't need the ones ended by "-build" or "-ex".
  4. Put the renamed ZIP on to  
    \\tiny.curamsoftware.com\ProductDevelopment\TechInf\Development_Tools\ibm_idx_source
\\tiny.curamsoftware.com\ProductDevelopment\TechInf\Development_Tools\ibm_idx_source
  5. Update the <JDE_ROOT>\devenv\setup\installers\ibm_idx_source.properties with the new version.
  6. Additionally, Put the new renamed ZIP to the <JDE_ROOT>\client\components\client-inf\src\main\javascript\internal and add into source control. Then update the the following to build scripts which reference the ZIP file in this location:<JDE_ROOT>\build_scripts\properties.xml                               <JDE_ROOT>\client\CoreInf\CuramCDEJ\bin\build-properties.xml
  7. Finally run "build cleanclient client" from <JDE_ROOT>. (If it fails at the setup stage, delete ibm_idx_source folder from devenv\tools and start build again)

Curam Widgets for External Container

The Curam widgets for the external container are based on the IDX toolkit. So we need to review them if they are stilll working. The following widgets should be reviewed and tested as shown the table below:

 
curam/app/ExternalApplication

curam/widget/form/DateTextBox
curam/widget/menu/MenuPane

IDX Themes for External Container

In the external container, we use a group of CSS files from IDX themes, they need to be reviewed as well. All of them is referenced in the file shown below:

 

<JDE_ROOT>\tests\TabTestApp\webclient\components\eclient\WebContent\CDEJ\jscript\ibmidxtk\idx\themes\oneui\oneui-curam-custom.css

Custom IDX Widget in JDE

As the out-of-the-box IDX widgets usually have some accessibility issues, it is likely that some IDX widget is customized so that these accessibility issues are resolved. Therefore in every IDX upgrade we need to ensure that these custom IDX widget is not affected by the upgrade.

Currently, we have the following custom IDX widgets in the Curam JDE:

 

curam/widget/IDXSelect
curam/widget/IDXFilteringSelect

For the details of how to review and upgrade the custom IDX widget, please refer to Custom Widget section in Dojo Upgrade Instructions

Testing

1. build the JDE, build TabTestApp.

2. run all automated tests, specifically check JavaScript ones

3. Verify overall functionality and look of the external application in TabTestApp.

4. Verify in a little more detail specific areas: external application banner, mega menu, person card, side menu ...

5. Specifically test that each of the Curam widgets for external container and custom IDX widgets in JDE are still working.
